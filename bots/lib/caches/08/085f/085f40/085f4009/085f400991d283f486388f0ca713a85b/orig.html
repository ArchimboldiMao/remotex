[北京/深圳/远程] BearyChat招 web 前端 / Clojure / 其他一些岗位

6个月前

这个招聘贴主要面向前端工程师和后端工程师，当然与此同时我们还有市场 /销售等岗位正在开放中。如果你对 SaaS 行业有些兴趣，喜欢不断尝试新的工具帮助提升效率，或者喜欢扁平轻松的团队环境，可以考虑继续把这个帖子读下去。

再次介绍一下产品和团队

我们在做一款叫做 BearyChat 的团队沟通工具，主要是提供一套信息流的梳理方案，并且通过开放接口（在我们产品内叫做机器人）把其他工具集成或者连接到聊天窗口中，帮助用户在团队沟通的过程当中一站式的实时接收各种工作工具推送过来的各种信息。

目前用户群体主要集中在北上广深等互联网行业最发达的城市和地区，在国内的同类产品中，目前我们的数据量是最好的并且远远领先。希望你加入之后这个用户地图和产品数据会变得更漂亮。

想要了解更多的话可以点这里浏览我们的官网。

团队的话，现在我们还是一个不到 20 人的小规模团队，创始人分别来自微软和阿里，其他成员也大多来自阿里 /网易 /豆瓣之类，不过说这些都是虚的。总之整个团队都比较年轻，互相之间可能比较容易有共同语言容易沟通。管理扁平、工程师文化之类的就不多说了。

目前我们在北京和深圳两个地方有办公室，还有一位同事在上海远程办公。所以如果你的坐标在北京或者深圳就最好了，不过如果的确十分合适的话，我们也可以考虑继续接受远程。

可能你会关注待遇的问题

  • 相对行业普遍水平有竞争力的薪酬，不会特别浮夸但是肯定要对得起你的能力

  • 五险一金，年终奖，调薪之类常规项目就不多展开了

  • 相对弹性的工作时间，双休（弹性≠加班）

  • 每月固定金额的饭补和房补

  • 每月一天年假，可以攒着用

  • 入职按需购买办公设备，现在除了运营妹为了轻便要求买 Mac air ，其他都是 Pro ，配外接显示器。如果你是 Linux/Windows 用户也是按需就好了

  • 有饮料零食之类不值得在招聘里提的东西，有不定期的 TB 和定期的团队旅行，其他内容你想到了可以提

  • 其他想到了再补充

关于前端和后端工程师

在这两个职位上，你的主要工作内容是和我们的产品团队（包括 PM 、设计师以及另外的工程师）一起实现用户所需的功能，以及保证用户在使用产品时有良好体验。

除此之外，在产品团队的每个人都有可能要承担某个功能的负责人角色，在需求得到充分讨论后确定要做之后，功能负责人要在团队里推进它继续前进，协调分工，对讨论中的分歧做出判断，并对最终效果进行衡量。

用户支持也是产品团队的职责之一，这是帮助我们时刻了解用户的需求和产品问题的最佳方式。

关于前端

我们现在的 web 应用主要基于 AngularJS (1.5) 开发，静态部署前端代码，并与后端通过 API 和 WebSocket 进行通信。

虽然对 AngularJS 的经验，不作为强制要求，但也需要你对至少一个主流的前端框架有使用经验，知道如何用他们解决常见的问题。

如果你有 Node-Webkit 和 Electron (Atom-shell) 的经验就最好不过了，我们还有一些基于 Web 的桌面客户端的维护工作需要进行。

关于后端

希望你熟悉 Clojure 编程以及 Clojure 下的 Web 框架 Ring, Compojure, sqlkorma 等。

同时具有优秀的数据库设计和优化能力, 至少精通一种数据库。

以下不作为强制要求，如果你能够满足或者部分满足则大大加分：

  • 熟悉 Java 及 JVM 性能调优

  • 有大规模网站服务器开发经验

  • 熟悉敏捷开发方法

  • 熟悉 RabbitMQ, ElasticSearch

前 /后端工程师的共同加分项

  • 发自内心的热爱技术，有探索精神，能从「做出对用户有用的产品」中获得快乐和成就感

  • 认同工具可以改善工作效率，乐于使用好工具，打磨自己使用的工具

  • 认同 Pull Request + Code Review 开发模式，有良好的 Git 使用习惯

  • 有 1 年以上在专业团队的工作经验，最近 1 年里在持续更新自己知识和技能

  • 有良好的团队合作精神, 有责任感, 有较好的沟通能力

  • 在代码层面，希望你能够在功能按时高质量交付的基础上，能注意代码可维护，可扩展性，能够遵循现有规范，并对其中不合理的地方加以改进

  • 在博客或 GitHub / twitter / stackoverflow 等等社区，有公开的内容，可以帮助我们全面了解到你

  • 坐标北京或者深圳（当然，特别合适我们也接受远程）
